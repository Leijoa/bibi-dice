# Steam 橫版 UI V2 配置草案

> 狀態：已停止作為主要方向。  
> 2026/05/19 起，Steam 版改採「桌面直式小遊戲」策略；此文件僅保留為橫版探索紀錄。新的主文件為 `STEAM_DESKTOP_PORTRAIT_STRATEGY.md`。

## 目標

V1 已經讓 Steam Demo 可以在桌面橫版中運作，但目前仍像是把手機版 UI 放大後重新排列。V2 的目標不是繼續塞更多元件，而是重新建立 PC / Steam Deck 玩家會自然閱讀的畫面重心。

本草案先處理三件事：

- 首頁按鈕與主視覺的關係。
- 戰鬥畫面中敵人、骰盤、分數、遺物與事件資訊的配置。
- 彈出訊息、傷害數字、牌型浮現、碎紙花等效果的定位規則。

## V1 問題整理

### 首頁

- 操作面板被放在右側，雖然可用，但和背景主視覺沒有形成好的構圖。
- 右側面板像是外掛在圖上的設定視窗，缺少遊戲標題畫面的儀式感。
- 目前的 UI 重心偏右，角色與骰子的視覺重心偏左，兩者沒有互相呼應。

### 戰鬥畫面

- 敵人血量放在左側大型欄位，畫面空間利用率不理想。
- 中央骰盤雖然清楚，但四周缺少戰場層次，導致大螢幕上顯得空。
- 右側分數欄目前偏功能列表，還沒有成為「戰鬥判讀區」。
- 手機版原本依賴小畫面密度，放大後通知、傷害、通關效果的座標都顯得飄。

### 動畫與彈出物

- Toast 訊息目前以整個 viewport 置中或堆疊，Steam 橫版時容易蓋住戰鬥重心。
- 敵人受傷數字依附在左側 enemy card，若敵人區重排，必須改為戰場錨點。
- 牌型浮現目前依骰盤中心計算，但 PC 版需要更明確的「骰盤上方演出區」。
- 碎紙花目前以全畫面 `origin` 為基準，Steam 版應改成依畫面事件位置決定。

## 首頁配置方案

### 方案 A：下方中央操作列

```text
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                  背景主視覺 / 角色 / 骰子                    │
│                                                              │
│                                                              │
│            [開始新遊戲] [新手教學] [玩法說明]                │
│            [靈魂奉獻] [收集冊] [歷史牌局] [設定]             │
│                         [語言]                               │
└──────────────────────────────────────────────────────────────┘
```

優點：
- 最接近傳統遊戲標題畫面，按鈕不會搶走主視覺。
- 按鈕集中在玩家視線自然落點，滑鼠與手把選單都好操作。
- Steam 商店截圖會比較像正式 PC 遊戲。

缺點：
- 若背景角色下半部資訊很重要，底部 UI 可能遮住圖。
- 需要替底部加深色漸層或遮罩，避免按鈕讀不清。

### 方案 B：左下角緊湊主選單

```text
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                 背景主視覺                                  │
│                                                              │
│  BIBI DICE                                                   │
│  [開始新遊戲]                                                │
│  [新手教學] [玩法說明] [收集冊] [設定]                       │
└──────────────────────────────────────────────────────────────┘
```

優點：
- 可以避開目前背景右側偏暗但角色不完整的區域。
- 適合讓標題與按鈕形成一個完整的主選單群。

缺點：
- 左側已經有角色手與骰子，容易和背景主動勢搶焦點。
- 若未補標題字樣，視覺會不夠穩。

### 方案 C：中央偏下主選單

```text
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│                  背景主視覺                                  │
│                                                              │
│                       BIBI DICE                              │
│                    [開始新遊戲]                              │
│              [新手教學] [玩法說明] [收集冊]                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

優點：
- 構圖最穩，玩家一眼知道這是主選單。
- 手把 / 鍵盤導覽最單純。

缺點：
- 容易遮住角色臉部或主視覺中心。
- 需要重看背景圖安全區。

### 首頁推薦

採用 **方案 A：下方中央操作列**。

理由：
- 目前背景圖的角色與骰子已經提供足夠的視覺吸引力，不需要再放一個右側大面板搶焦點。
- Steam 頁面第一張圖通常要快速傳達「這是可玩的 PC 遊戲」，底部中央主選單最直覺。
- 後續若加入手把操作，底部操作列也最容易做 focus ring 與選單順序。

## 戰鬥畫面配置方案

### 方案 A：上方敵人舞台 + 中央骰盤 + 右側判讀欄

```text
┌──────────────────────────────────────────────────────────────┐
│ Top Bar：BIBI DICE / 關卡 / HP / 設定 / 回標題 / 牌型表       │
├──────────────────────────────────────────────────────────────┤
│ Enemy Stage：敵人名稱 / HP Bar / 回合 / 枷鎖 / 傷害數字       │
├───────────────────────────────┬──────────────────────────────┤
│ Dice Board                     │ Battle Readout                │
│ 4 x 2 骰子                     │ 預估傷害                      │
│ 重骰 / 攻擊                    │ A/B/C/D 牌型                  │
│ 牌型浮現區                     │ 遺物觸發 / 事件 Log           │
└───────────────────────────────┴──────────────────────────────┘
```

優點：
- 敵人血量回到「上方戰場」位置，比左側欄更像 PC 戰鬥 UI。
- 中央下方專注在骰盤與操作，右側負責判讀，不互相干擾。
- 傷害數字、敵人受擊、碎紙花都有明確的上方戰場錨點。

缺點：
- 需要把現有 `.card-enemy` 從左欄改成橫向舞台，CSS 變動較大。
- 若未來有敵人立繪，要預留敵人舞台高度。

### 方案 B：左側戰鬥舞台 + 中央骰盤 + 右側資訊

```text
┌──────────────────────────────────────────────────────────────┐
│ Top Bar                                                       │
├───────────────────────┬──────────────────────┬───────────────┤
│ Enemy Stage            │ Dice Board            │ Readout       │
│ 敵人 / HP / 傷害       │ 4 x 2 骰子 / 操作     │ 牌型 / 遺物   │
│ 事件 Log               │ 牌型浮現              │               │
└───────────────────────┴──────────────────────┴───────────────┘
```

優點：
- 接近 V1，實作風險低。
- 左側可以放敵人立繪、受擊、回合事件。

缺點：
- 使用者已經明確覺得敵人血量在左邊不滿意。
- 大螢幕仍可能形成左側空欄問題。

### 方案 C：中央敵人 + 下方骰盤 + 左右輔助欄

```text
┌──────────────────────────────────────────────────────────────┐
│ Top Bar                                                       │
├──────────────┬──────────────────────────────┬───────────────┤
│ Relics       │ Enemy Stage / HP / Damage     │ Score         │
│              │ Dice Board                    │ Hand Types    │
│              │ Controls                      │ Event Log     │
└──────────────┴──────────────────────────────┴───────────────┘
```

優點：
- 敵人與骰盤都在中央，戰鬥感最強。
- 左右欄可以分別成為「長期能力」與「本回合判讀」。

缺點：
- 中央欄會非常忙，需要更大幅度重排 HTML / CSS。
- Steam Deck 1280x800 容易擠壓。

### 戰鬥推薦

採用 **方案 A：上方敵人舞台 + 中央骰盤 + 右側判讀欄**。

理由：
- 解決「敵人血量在左邊不自然」的核心問題。
- 傷害、受擊、通關、枷鎖等演出都能集中在上方戰場，不會到處亂飛。
- 中下方保留骰盤主操作，右側維持玩家判讀需要的資訊密度。

## V2 戰鬥資訊配置

### Top Bar

保留，但減少視覺重量。

內容：
- BIBI DICE
- 關卡 / 層數
- HP
- 設定
- 回標題
- 牌型表

規則：
- 高度 56px 至 64px。
- 不承擔主要戰鬥資訊，只做全域狀態與導航。

### Enemy Stage

新的上方戰場區，取代 V1 左側敵人欄。

內容：
- 敵人名稱。
- 敵人 HP bar。
- HP 數字。
- 剩餘回合。
- 枷鎖 badge。
- 敵人受擊傷害數字。
- 敵人擊敗 / Boss 特效錨點。

建議尺寸：
- PC：高度 150px 至 190px。
- Steam Deck：高度 120px 至 150px。

### Dice Board

中下方主要操作區。

內容：
- 4 x 2 骰子。
- 重骰 / 攻擊按鈕。
- 牌型浮現區。
- 骰子鎖定提示。

建議尺寸：
- PC 骰子：104px 至 128px。
- Steam Deck 骰子：78px 至 96px。

### Battle Readout

右側判讀欄，不只是分數欄。

內容：
- 預估傷害。
- A/B/C/D 牌型結果。
- 已有遺物。
- 本回合觸發紀錄。
- 敵人特殊規則摘要。

資訊密度規則：
- 預估傷害置頂，字級最大。
- A/B/C/D 牌型其次，做成 2 x 2 compact cards。
- 遺物與事件 log 分成 tabs 或上下兩段，不要同時無限展開。

## 彈出物與動畫定位規則

### Toast 訊息

目前：全畫面固定置中堆疊。

V2：
- 改成 `body.steam-layout` 下固定到右下或右側判讀欄上方。
- 不遮住骰盤與敵人舞台。
- 多則訊息垂直堆疊，寬度限制在 360px 至 460px。

建議錨點：
- PC：右側判讀欄內上方或右下角。
- Steam Deck：右上角，最多顯示 2 則，其餘排隊。

### 敵人扣血數字

目前：依附 `.card-enemy` 內的 `#damage-container`。

V2：
- `#damage-container` 改成 Enemy Stage 內的絕對定位層。
- 傷害數字從 HP bar 或敵人名稱附近往上飄。
- 若未來加入敵人立繪，改從敵人立繪中心偏上飄。

建議錨點：
- `enemy-stage-center`
- `enemy-hp-bar-center`

### 牌型浮現

目前：依 `#dice-container` 中心計算。

V2：
- 保留骰盤錨點，但浮現位置改到骰盤上方 20px 至 60px。
- 多個牌型由中間向左右散開，不遮住骰子本體。
- Mythic / Legendary 可以短暫跨到 Enemy Stage 與 Dice Board 之間，形成攻擊預告。

建議錨點：
- `dice-board-center`
- `dice-board-top`

### 通關碎紙花

目前：全畫面 confetti，origin 固定。

V2：
- 小怪擊敗：從 Enemy Stage 中央偏上爆出，粒子量少。
- Boss 擊敗：從 Enemy Stage 兩側與畫面上緣爆出，粒子量中。
- 全破：全畫面 confetti，允許覆蓋主畫面。

建議錨點：
- `enemy-stage-center`
- `viewport-top-left`
- `viewport-top-right`

### 商店 / 獎勵彈窗

V2 商店應該像橫版中場選擇畫面，不是手機彈窗放大。

配置：
- 左側 2/3：商品或獎勵卡 grid。
- 右側 1/3：玩家狀態、目前遺物、金錢 / 靈魂。
- 底部：刷新 / 下一關。

## Phase 2 實作順序

### Phase 2A：首頁重排

涉及檔案：
- `css/style.css`
- 視需要微調 `index.html`

工作：
- 將首頁從右側面板改成下方中央操作列。
- 保留背景主視覺安全區。
- 驗證 1920x1080 與 1280x800。

### Phase 2B：戰鬥 V2 骨架

涉及檔案：
- `css/style.css`
- 視需要微調 `index.html`

工作：
- 將敵人區從左側欄改成上方 Enemy Stage。
- 中下方放 Dice Board。
- 右側改為 Battle Readout。
- 保留現有 JS id，不改 engine 與 data。

### Phase 2C：彈出物定位

涉及檔案：
- `css/style.css`
- `js/ui.js`

工作：
- Steam layout 下重新定位 toast。
- 傷害數字改依 Enemy Stage 錨點。
- 牌型浮現改依 Dice Board top 錨點。
- confetti 依事件類型使用不同 origin。

### Phase 2D：截圖驗證

涉及檔案：
- `promo/steam/assets/`
- `promo/steam/ASSET_CHECKLIST.md`

工作：
- 重抓 5 到 6 張 1920x1080 Steam 實機截圖。
- 重新檢查商店頁可用截圖。
- 確認戰鬥、商店、Boss、靈魂奉獻至少各一張。

## 推薦決策

V2 採用：

- 首頁：下方中央操作列。
- 戰鬥：上方 Enemy Stage + 中下 Dice Board + 右側 Battle Readout。
- 彈出物：依 Enemy Stage、Dice Board、Battle Readout 建立固定錨點。

這個方向會比 V1 更像真正 PC / Steam 橫版 UI，也能解決目前「畫面空、位置怪、敵人血量不自然」三個核心問題。
