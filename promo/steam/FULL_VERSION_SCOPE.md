# Demo 與正式版承諾範圍

## 原則

Steam 商店頁只能承諾玩家可實際取得的內容。Demo 頁面只寫 Demo 內玩得到的內容；正式版頁面可以寫正式版方向，但未完成內容必須保持清楚、保守，不寫未定案數量。

## Demo 目前可對外描述

Demo 可定位為：

> 以完整核心玩法為主的免費測試版，包含一輪主要挑戰、Boss 枷鎖、無限塔與局外靈魂奉獻。

Demo 已包含：

- 8 顆骰子的牌型判定與傷害倍率計算。
- 有限重骰次數與有限回合的戰鬥節奏。
- 10 關主要流程。
- 最終 Boss。
- Boss 枷鎖限制。
- 無限塔。
- 遺物取得與構築。
- 消耗品。
- 融合遺物與融合素材邏輯。
- 靈魂奉獻局外成長。
- 收集冊。
- 歷史牌局。
- 教學與玩法說明。
- 音效與 BGM。
- 繁體中文、簡體中文、英文、日文。

Demo 可用賣點：

- 短局制，容易重開。
- 骰面保留與重骰選擇帶來風險判斷。
- 遺物與融合會讓分數爆發路線差異很大。
- 通關後仍可用無限塔測試構築上限。
- 靈魂奉獻提供跨局成長。

Demo 不應承諾：

- Steam 成就。
- Steam Cloud。
- Steam Deck Verified。
- 控制器完整支援。
- 線上排行榜。
- 每日挑戰。
- 正式版新增內容的精確數量。
- 尚未完成的新模式、新 Boss、新劇情或新系統。

## 正式版建議承諾

正式版可以承諾的方向：

- 保留 Demo 的核心玩法。
- 擴充可重玩內容。
- 加入 Steam 成就。
- 擴充靈魂奉獻項目。
- 增加更多遺物與融合方向。
- 增加更多敵人、Boss 或挑戰條件。
- 持續調整平衡、教學與操作體驗。

正式版文案建議寫法：

> 正式版將在 Demo 的核心玩法上擴充更多構築內容，包含 Steam 成就、更豐富的靈魂奉獻項目，以及更多遺物、融合方向與挑戰條件。

這種寫法可以表達方向，但不會把尚未完成的精確數量變成硬承諾。

## 正式版暫不承諾

除非後續已實作或明確排程，正式版商店頁暫不寫：

- 具體成就數量。
- 具體遺物數量。
- 具體 Boss 數量。
- 完整劇情章節。
- 排行榜。
- 每日挑戰。
- 模組或工作坊。
- 多人遊戲。
- Steam Deck Verified。
- 雲端存檔。

## 成就策略

Steam 官方建議 Demo 不啟用成就。正式版可加入成就，Demo 玩家若已完成特定條件，可在正式版讀取存檔或記錄後再補發對應成就。

建議正式版成就方向：

- 首次通關。
- 首次擊敗 Boss。
- 首次進入無限塔。
- 首次獲得融合遺物。
- 單次傷害達到指定門檻。
- 使用特定牌型擊敗敵人。
- 靈魂奉獻達到指定階段。
- 收集冊完成度達到指定比例。

目前不要承諾成就數量，等正式版成就系統設計完成後再填。

## 靈魂奉獻擴充方向

已存在項目可作為 Demo 內容描述：

- 最大 HP。
- 商店折扣。
- 初始金幣。
- 重骰次數。
- 初始遺物。
- 最終傷害或靈魂爆發相關成長。

正式版可擴充方向：

- 更多開局變化。
- 更多局外風險與回報選擇。
- 讓無限塔與高難度挑戰更有長期目標。
- 與成就或收集冊形成更清楚的長線追求。

正式版商店頁建議只寫「更豐富的靈魂奉獻項目」，不要先寫具體名稱。

## Demo 與正式版頁面切分

### Base Game Coming Soon 頁

主目標：

- 蒐集願望清單。
- 說清楚正式版方向。
- 放 Demo 下載入口。

可寫：

- Demo 現有內容。
- 正式版將擴充內容量。
- 正式版會加入 Steam 成就與更多靈魂奉獻項目。

### Demo 頁

主目標：

- 讓玩家知道 Demo 下載後能玩到什麼。
- 不讓玩家誤以為 Demo 含正式版所有功能。

可寫：

- Demo 內含一輪主要挑戰、Boss、無限塔、靈魂奉獻。
- Demo 是免費測試版。
- 正式版頁面可加入願望清單。

不可寫：

- Demo 內沒有的 Steam 功能。
- 正式版才會加入的內容，除非明確標為正式版內容。

## 目前最小可行 Steam 上架順序

1. 建立 Steam Direct app。
2. 建立 Base Game Coming Soon 頁。
3. 準備至少 5 張實機截圖與必要 capsule。
4. 上傳目前 `dist/steam-demo` 作為 Demo build。
5. 建立 Demo app 並關聯 Base Game。
6. 送 Base Game store presence 審核。
7. Coming Soon 頁公開至少 2 週。
8. Demo build 與 Demo store presence 審核。
9. 發布 Demo，並確認 Base Game 頁面顯示 Download Demo。
