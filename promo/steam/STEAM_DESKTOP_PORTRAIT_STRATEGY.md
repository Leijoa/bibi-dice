# Steam 桌面直式小遊戲策略

## 定位

BIBI DICE 的 Steam 版不再朝橫版大螢幕 UI 前進。新的方向是桌面直式小遊戲：

- 啟動後是一個固定直式比例的桌面視窗。
- 玩家可以在設定或啟動選項中切換視窗大小，但比例維持直式。
- 不承諾 Steam Deck / 掌機相容。
- 商業定位是低價、輕量、短局、可反覆遊玩的桌面小品。
- 建議售價區間：US$2.99。

## 為什麼改方向

原本的遊戲節奏是空閒時玩一兩下，不是長時間坐在大螢幕前操作的大型 Roguelite。手機直式 UI 已經符合遊戲節奏，硬改成橫版會帶來三個問題：

- 需要重做大量 UI，成本高。
- 畫面被迫填滿大螢幕，容易空洞。
- 玩家會期待更大的內容量與掌機支援，和 US$2.99 的小品定位不一致。

桌面直式版則保留原本優勢：

- 現有 UI 可延續。
- 截圖與商店文案可以明確說明這是輕量桌面遊戲。
- Steam 版工程重點轉為包裝、解析度、離線、存檔與操作體驗。

## 視窗尺寸

基準比例維持 `9:16`，以現有 `450 x 800` 為設計基準。

建議支援三種視窗大小：

- 小：`450 x 800`
- 中：`540 x 960`
- 大：`675 x 1200`

Steam 桌面版預設使用 `540 x 960`。  
若玩家螢幕高度不足，啟動時降到 `450 x 800`。

## 不支援項目

- 不支援橫版 UI。
- 不承諾 Steam Deck 驗證。
- 不做 1920x1080 全畫面戰鬥 UI。
- 不因 Steam 版而重寫核心戰鬥介面。

## Steam 商店呈現

Steam 規定截圖仍可用 16:9 圖片呈現，但內容不必假裝是橫版遊戲。

推薦截圖方式：

- 1920x1080 畫布中央放直式遊戲視窗。
- 背景使用遊戲主視覺或暗色模糊背景。
- 旁邊可以放局部放大細節，例如骰子、遺物、牌型結果。
- 截圖文案避免暗示大螢幕橫版體驗。

建議截圖主題：

- 標題畫面直式視窗。
- 戰鬥開局。
- 牌型倍率與傷害爆發。
- 遺物商店。
- Boss / 枷鎖。
- 靈魂奉獻或收集冊。

## 技術方向

### 目前短期

- Steam Demo build 注入 `body.steam-portrait`。
- 保留 450x800 的遊戲容器與現有縮放邏輯。
- 在大螢幕 web 預覽時限制最大放大倍率，避免畫面被拉得太大。

### 正式包裝

建議優先評估 Electron，因為目前專案是 Vanilla JS / HTML / CSS，接入成本最低。

Electron 需求：

- 固定視窗比例。
- 預設 `540 x 960`。
- 最小 `450 x 800`。
- 可選 `675 x 1200`。
- 關閉外部網路依賴。
- 儲存資料沿用 localStorage，後續再評估 Steam Cloud。

Tauri 也可行，但初期會增加 Rust 工具鏈成本。除非明確需要更小包體，否則先用 Electron 較務實。

## 實作階段

### Phase P1：Steam Portrait Build

涉及檔案：

- `scripts/publish-steam-demo.ps1`
- `css/style.css`
- `js/ui.js`

工作：

- 將 Steam build class 從 `steam-layout` 改為 `steam-portrait`。
- 保留直式比例與手機版 UI。
- 限制 Steam portrait 模式最大縮放。
- 確認 `dist/steam-demo` 不再套用橫版三欄 CSS。

### Phase P2：桌面視窗包裝

涉及檔案：

- `package.json`
- 新增 Electron 入口檔
- 新增 build script

工作：

- 建立桌面 App 殼。
- 設定預設視窗 `540 x 960`。
- 加入解析度切換的基礎機制。

### Phase P3：Steam 商店素材重做

涉及檔案：

- `promo/steam/assets/`
- `promo/steam/ASSET_CHECKLIST.md`

工作：

- 重做 1920x1080 截圖，呈現直式桌面視窗。
- 更新素材檢查表，標記橫版戰鬥截圖廢止。

### Phase P4：正式版承諾範圍調整

涉及檔案：

- `promo/steam/FULL_VERSION_SCOPE.md`
- `promo/steam/STEAM_STORE_BRIEF.md`

工作：

- 將文案定位改為 US$2.99 桌面小品。
- 不承諾掌機支援。
- 正式版新增內容集中在成就、靈魂奉獻、遺物、牌型與短局重玩性。

## 目前決策

採用桌面直式小遊戲方向。  
橫版 UI 規格與 V2 layout 保留為探索紀錄，不再作為主要實作路線。
