# Steam 橫版 UI 規格草案

## 目標

Steam 版應提供真正適合 PC 與 Steam Deck 的橫版介面，而不是把手機直式 UI 放進寬螢幕中。手機版與 itch.io 版可以保留現有直式體驗；Steam 版新增橫版版型，讓玩家在桌面、筆電與 Steam Deck 上看到完整且穩定的橫向布局。

## 設計原則

- 不改核心玩法邏輯，`engine.js` 與 `data.js` 不應因橫版 UI 需求而改動。
- 優先用 CSS 與少量 HTML 容器調整完成，不重寫整個 UI。
- 保留現有手機直式版型，Steam 橫版以獨立 class 或 build flag 啟用。
- Steam 截圖、Steam Deck 與 PC 實際遊玩都以橫版版型為主。
- UI 必須支援滑鼠點擊、鍵盤快捷鍵與未來控制器焦點導覽。
- 不以「放大手機畫面」為目標，而是重新分配資訊區域。

## 參考稿採用與不採用項目

使用者提供的兩份橫版參考稿可作為 Steam 版 UI 的結構方向，但不直接照搬。它們的價值在於「資訊區域重新分配」，不是視覺素材或技術棧。

### 採用

- **固定 Top Bar**：保留遊戲名、關卡、設定、回標題與說明入口，讓 PC 玩家一眼知道目前狀態。
- **戰鬥三欄結構**：左側敵人與枷鎖，中間骰盤與操作，右側牌型、傷害拆解與遺物。
- **中央 4 x 2 骰盤**：橫版時骰子應是主視覺焦點，尺寸比手機版更大、更清楚。
- **傷害預覽獨立成區**：參考稿把 estimated damage 放在骰盤上方，這符合 Steam 版的資訊掃讀方式。
- **商店商品 Grid**：Steam 版商店應使用 2 x 2 或 3 x 2 商品卡片，而不是手機版長列表。
- **商店底部行動列**：重新整理商店與前往下一關應固定在底部，避免橫版中操作按鈕失焦。
- **右側玩家狀態欄**：商店畫面可保留金幣、HP、已持有遺物，方便購買時比較。
- **明確的枷鎖提示**：Boss / 枷鎖在橫版畫面中要成為可讀的狀態 badge。

### 不採用

- **完整左側 SideNav**：目前遊戲是短局制 Roguelite，不需要大型後台式側邊導航；照搬會吃掉太多橫向空間。
- **Material Symbols 與 Google Fonts CDN**：Steam build 必須離線可執行，不能引入外部字型或 icon CDN。
- **參考稿中的 AI 商品大圖**：目前遊戲沒有每個遺物的本地插畫資產，第一版商店卡片以稀有度色塊、符號與文字資訊為主。
- **英文固定文字**：正式 UI 仍需遵守現有 i18n 架構，不能在 HTML / JS / CSS 中硬編碼新增顯示文字。
- **過度 dashboard 化**：BIBI DICE 需要保留紫黑、骰子、鎖鏈、奇幻賭局感，不要變成通用 SaaS 工具介面。
- **大面積玻璃擬態與重模糊**：可保留輕量透明感，但過多 backdrop blur 會影響低階 PC 與 Steam Deck 效能。

### Phase 1 取用方式

Phase 1 只取結構，不做完整視覺重製：

- 新增 Steam 橫版 layout 開關。
- 讓戰鬥畫面形成左敵人、中骰盤、右分數遺物的骨架。
- 讓商店從手機列表改為橫版商品 grid 與底部操作列。
- 不新增外部 icon、字型或商品插圖。
- 不改核心邏輯，不改資料層，不改語系 key。

## 啟用方式

建議新增一個明確的 Steam 版樣式開關：

```html
<body class="steam-layout">
```

或在 Steam build 輸出時由腳本注入：

```html
<html data-build="steam-demo">
```

建議第一階段使用 `body.steam-layout`，因為 CSS 選擇器清楚，也不影響目前 itch/mobile 版。

## 目標解析度

### PC 桌面主要目標

- `1920 x 1080`
- `1600 x 900`
- `1366 x 768`

### Steam Deck 主要目標

- `1280 x 800`
- 16:10 橫版

### 最低可接受橫版

- `1024 x 640`
- 若低於此寬度，回退到現有手機直式或緊湊版型。

## 版面總覽

### 戰鬥畫面

建議改為三欄式：

```text
┌──────────────────────────────────────────────────────────────┐
│ Header：遊戲名 / 關卡 / HP / 設定 / 回標題                    │
├───────────────┬───────────────────────────┬──────────────────┤
│ Enemy Panel   │ Dice Board + Controls      │ Score / Relics   │
│ 敵人資訊      │ 骰盤與操作                 │ 分數與遺物       │
│ HP / 回合     │ 8 顆骰子 + 重骰/攻擊       │ 牌型拆解         │
│ 枷鎖提示      │ 傷害預覽                   │ 已持有遺物       │
└───────────────┴───────────────────────────┴──────────────────┘
```

建議欄寬：

- 左欄敵人資訊：`280px` 到 `360px`
- 中欄骰盤與操作：彈性寬度，最小 `520px`
- 右欄分數與遺物：`360px` 到 `460px`

在 `1920 x 1080`：

- 左欄約 `340px`
- 中欄約 `820px`
- 右欄約 `460px`
- 外框最大寬度約 `1680px` 到 `1760px`

在 `1280 x 800`：

- 左欄約 `260px`
- 中欄約 `520px`
- 右欄約 `340px`
- 減少 padding、縮小標題字級與卡片間距。

## 區塊規格

### Header

保留：

- `BIBI DICE`
- 關卡資訊
- HP
- 設定
- 回標題

調整：

- Header 高度控制在 `56px` 到 `72px`。
- 關卡資訊與 HP 必須在同一列可讀。
- Steam Deck 上按鈕需至少 `44px` 高，方便觸控。

### Enemy Panel

放在左欄，內容包含：

- 敵人名稱
- 關卡 / 層數徽章
- HP bar
- HP 數字
- 剩餘回合
- 當前枷鎖 badge
- Boss 或菁英視覺提示

行為：

- 敵人名稱可單行截斷，不撐爆布局。
- 枷鎖 badge 可點擊開說明。
- HP bar 在 1280 寬度仍需清楚。

### Dice Board

放在中欄，為主要視覺焦點。

建議：

- 8 顆骰子維持 `4 x 2` 或 `8 x 1` 的清楚排列。
- 桌面版優先 `4 x 2`，Steam Deck 也維持 `4 x 2`。
- 每顆骰子尺寸：
  - PC：`92px` 到 `120px`
  - Steam Deck：`70px` 到 `88px`
- 骰子鎖定狀態必須比手機版更明顯，可加外框、鎖圖示或亮度差。

操作按鈕：

- 重骰與攻擊按鈕放在骰盤右側或下方。
- PC 版可右側直排。
- Steam Deck 版建議下方橫排，避免右側太窄。

### Damage Preview

目前 `final-damage-preview` 可改放中欄上方或中欄下方。

建議：

- 在 PC 版放於骰盤上方，讓玩家先看到本回合預估傷害。
- 在 Steam Deck 版放於骰盤下方，避免壓縮敵人資訊。
- 數字要大，但不要使用手機版那種過度吃高度的尺度。

### Score / Relics Panel

放在右欄，內容包含：

- A/B/C/D 區牌型拆解
- 目前最終傷害
- 已持有遺物
- 遺物橫向或網格列表

調整：

- 右欄應該是可掃讀資訊欄，不要跟骰盤搶主視覺。
- 牌型區塊固定高度，內容過多時內部滾動。
- 已持有遺物建議改成 `grid` 或可換行 chip，而不是只靠橫向滑動。
- 每個遺物 chip 仍可點擊開說明。

## 商店畫面

商店不應全螢幕只做手機式列表。橫版建議：

```text
┌──────────────────────────────────────────────────────────────┐
│ 商店標題 / 金幣 / 下一關資訊                                  │
├───────────────────────────────────────────────┬──────────────┤
│ 商品卡片 grid                                  │ 玩家狀態     │
│ 2 x 2 或 3 x 2                                 │ HP / 金幣    │
│ 遺物、消耗品、融合提示                         │ 已持有遺物   │
├───────────────────────────────────────────────┴──────────────┤
│ 重新整理商店 / 離開並前往下一關                                │
└──────────────────────────────────────────────────────────────┘
```

PC：

- 商品卡片可 `3 x 2`。
- 右側保留玩家狀態與已持有遺物。

Steam Deck：

- 商品卡片可 `2 x 2`，必要時垂直滾動。
- 底部操作列固定，避免「下一關」按鈕被推到看不到。

## Modal 規格

適用：

- 靈魂奉獻
- 收集冊
- 歷史牌局
- 玩法說明
- 遺物說明
- 枷鎖說明

橫版規格：

- 最大寬度不超過畫面 `80vw`。
- 最大高度不超過 `86vh`。
- Steam Deck 上最大寬度可放到 `92vw`。
- 標題列固定，內容區滾動。
- 關閉按鈕至少 `44px x 44px`。

靈魂奉獻：

- PC 版可做 `2` 欄升級卡片。
- Steam Deck 版可維持單欄或 `2` 欄緊湊卡片。

收集冊：

- PC 版適合更多欄 grid。
- Steam Deck 版以可讀性優先，不要塞太多小字。

## 驗收標準

PC `1920 x 1080`：

- 戰鬥畫面不需捲動即可看到敵人、骰盤、操作、分數與遺物。
- 商店畫面不需捲動即可看到至少 4 個商品與主要操作按鈕。
- 所有主要文字不重疊、不截斷到不可理解。

Steam Deck `1280 x 800`：

- 戰鬥畫面可完整操作。
- 重骰與攻擊按鈕一眼可見。
- 字級不低於 `12px`，主要操作不低於 `14px`。
- 主要按鈕高度不低於 `40px`，重要按鈕建議 `44px`。

截圖：

- 6 張 Steam screenshot 全部為 `1920 x 1080`。
- 戰鬥、商店、Boss 畫面皆使用橫版 UI。
- Playwright 驗證無 console error、page error、failed request。

## 風險與注意

- 現有 UI 有大量 Tailwind utility class，純 CSS override 可能需要提高選擇器權重。
- `#game-scaler` 目前是手機版縮放核心，Steam 橫版必須小心隔離，不要破壞手機版。
- `score-area` 與 `battle-area` 目前已在 `lg:flex-row` 做過橫排，但資訊比例不適合 Steam，需要重新定義寬度與高度。
- 商店 overlay 目前蓋滿 `main-container`，橫版需補足右側玩家狀態或至少讓商品 grid 更像桌面介面。
- 未來若加入控制器支援，需要再補焦點順序與快捷鍵提示，但第一階段先不承諾。
